# Live Observation 임시 검증 사이트 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Each task uses TDD and an independent review gate.

**Goal:** CloudFront → ALB → EC2 요청의 성공 receipt를 SketchCatch collector와 SSE까지 연결하는 임시 Live Observation PoC를 만든다.

**Architecture:** S3 origin의 정적 audience 페이지가 CloudFront의 기본 behavior로 제공되고, /api/traffic만 ALB origin으로 라우팅된다. EC2 Fastify handler가 생성한 receipt를 browser가 capability 인증 collector로 전달하면, SketchCatch API가 중복 제거한 snapshot을 SSE로 발표 화면에 발행한다.

**Tech Stack:** Node.js 22, TypeScript, Fastify 5, 기존 node:test, 정적 HTML/CSS/ES module JavaScript, Terraform, k6, AWS S3/CloudFront/ALB/EC2 Auto Scaling.

## Global Constraints

- LIVE_OBSERVATION_POC_ENABLED 기본값은 정확히 false다.
- DB, Redis, Docker, ECS, Route 53, 커스텀 도메인, QR UI, presenter boost, CloudWatch polling은 이 PoC에 추가하지 않는다.
- capability와 secret은 환경 변수/secret store에서만 읽고 저장·로그·URL query·browser persistent storage에 남기지 않는다.
- public collector body limit은 정확히 1 KiB이며 CORS는 exact audience origin만 허용한다.
- session은 생성 시점부터 정확히 15분이고 어떤 요청도 TTL을 연장하지 않는다.
- 실제 AWS apply, destroy, k6 부하는 사용자 명시 승인 없이는 실행하지 않는다.

## 1. 목적과 완료 정의

이 PoC는 CloudWatch polling, Redis, 실제 scale-out 완료 판정보다 먼저 아래의 빠른 성공 이벤트 경로를 실제로 연결하는 것이 목적이다.

~~~
Audience browser button
  → CloudFront /api/traffic
  → ALB
  → Target Group
  → EC2 Fastify traffic handler
  → 2xx JSON receipt
  → Audience browser collector POST
  → SketchCatch API receipt dedupe/count
  → SketchCatch SSE snapshot
  → Presenter/monitor 화면 갱신
~~~

완료 기준:

1. S3 origin의 audience 정적 페이지가 CloudFront HTTPS URL에서 열린다.
2. 버튼 클릭이 CloudFront의 /api/traffic behavior를 거쳐 ALB와 EC2 Fastify handler에 도달한다.
3. EC2 handler가 발급한 eventId, instanceId, receivedAt을 포함한 2xx JSON을 받는다.
4. audience browser는 유효한 2xx receipt가 있을 때만 SketchCatch public collector를 호출한다.
5. SketchCatch collector는 capability와 eventId를 검증하고 중복 event를 한 번만 세며 SSE snapshot을 발행한다.
6. SSE 구독자는 acceptedEventCount와 lastReceipt.instanceId 변경을 받는다.
7. k6 부하 테스트가 같은 receipt 흐름을 반복하고 ASG desiredCapacity가 1에서 2로 증가하는 것을 수동 확인할 수 있다.

## 2. 비목표와 안전 경계

- Redis, RDS 저장, QR UI, presenter boost, CloudWatch polling, AWS observer, 실제 scale-out 완료 판정은 이 PoC 범위 밖이다.
- collector 상태는 process memory에만 저장한다. LIVE_OBSERVATION_POC_ENABLED=true일 때도 collector는 단일 API process에서만 실행한다.
- 이 PoC의 browser receipt는 브라우저 보고 성공이다. eventId가 EC2 handler에서 발급되어 실제 요청 성공과 연결되지만, capability를 가진 browser가 event를 위조하지 못한다는 암호학적 증명까지는 하지 않는다.
- UI와 로그는 이 이벤트를 AWS 실측 또는 확장 완료로 표현해서는 안 된다.
- 실제 고객 Deployment, 기존 Live Observation route, Terraform execution worker를 변경하지 않는다.
- terraform apply, terraform destroy, 실제 k6 부하 실행은 사용자 명시 승인 없이 실행하지 않는다.
- secret, capability, Authorization header, cookie, 실제 AWS account ID, 실제 URL fragment를 로그, 문서, 테스트 fixture, screenshot, artifact에 넣지 않는다.

## 5. 고정 HTTP 및 DTO 계약

### 5.1 EC2 traffic route

~~~
GET /health
~~~

응답:

~~~json
{ "status": "ok" }
~~~

~~~
POST /api/traffic
~~~

성공 응답:

~~~http
HTTP/1.1 200 OK
Cache-Control: no-store
Content-Type: application/json
~~~

~~~ts
type TrafficReceipt = {
  eventId: string;
  instanceId: string;
  receivedAt: string;
};
~~~

규칙:

- eventId는 EC2 Fastify handler가 randomUUID()로 생성한다.
- receivedAt은 browser 시간이 아니라 EC2 application 시간이다.
- instanceId는 INSTANCE_ID 환경 변수를 우선 사용한다. 실제 EC2에서는 IMDSv2를 한 번 읽어 cache하고, local test fallback은 local-dev다.
- request header/body를 receipt에 반사하지 않는다.
- CloudFront가 응답을 cache하지 않도록 Terraform cache policy와 응답 header를 모두 설정한다.

### 5.2 SketchCatch session create route

~~~
POST /api/live-observation-poc/sessions
Authorization: Bearer <existing SketchCatch access token>
~~~

이 route는 기존 SketchCatch 인증/프로젝트 ownership pattern을 사용한다. 테스트를 위해 public으로 열지 않는다.

~~~ts
type CreateLiveObservationPocSessionResponse = {
  observationId: string;
  createdAt: string;
  expiresAt: string;
  audienceUrl: string;
  streamUrl: string;
};
~~~

audienceUrl 형식:

~~~
https://<cloudfront-domain>/#observationId=<uuid>&collector=<encoded-api-origin>&capability=<kid.token>
~~~

capability는 query string이나 server log에 넣지 않는다. fragment는 HTTP request와 Referer에 포함되지 않으며, audience page는 읽은 뒤 즉시 주소창에서 제거한다.

### 5.3 Public collector route

~~~
OPTIONS /api/live-observation-poc/public/:observationId/events
POST    /api/live-observation-poc/public/:observationId/events
Authorization: LiveObservation <kid>.<token>
Content-Type: application/json
~~~

요청 body:

~~~ts
type CollectLiveObservationPocEventRequest = {
  eventId: string;
  instanceId: string;
  receivedAt: string;
};
~~~

응답:

~~~ts
type CollectLiveObservationPocEventResponse = {
  accepted: boolean;
  acceptedEventCount: number;
};
~~~

PoC store 내부 타입은 아래처럼 고정한다.

~~~ts
type LiveObservationPocSession = {
  observationId: string;
  status: "active" | "expired";
  capabilityKid: string;
  tokenVersion: 1;
  createdAt: string;
  expiresAt: string;
  acceptedEventCount: number;
  lastReceipt: TrafficReceipt | null;
};

type LiveObservationPocSnapshot = {
  observationId: string;
  status: "active" | "expired";
  acceptedEventCount: number;
  lastReceipt: TrafficReceipt | null;
  observedAt: string;
};
~~~

| 조건 | 상태 | body 핵심 |
| --- | --- | --- |
| 새 receipt | 202 | accepted: true |
| 같은 eventId 재전송 | 200 | accepted: false |
| Authorization/capability 오류 | 401 | public generic error |
| observationId 없음 | 404 | public generic error |
| 세션 만료/종료 | 410 | public generic error |
| schema 오류 | 400 | public generic error |
| body 1 KiB 초과 | 413 | public generic error |

### 5.4 SSE route

~~~
GET /api/live-observation-poc/sessions/:observationId/stream
Accept: text/event-stream
Authorization: Bearer <existing SketchCatch access token>
~~~

SSE event:

~~~text
event: snapshot
data: {"observationId":"...","status":"active","acceptedEventCount":1,"lastReceipt":{"eventId":"...","instanceId":"...","receivedAt":"..."},"observedAt":"..."}
~~~

SSE는 collector가 새 receipt를 accepted했을 때만 발행한다. duplicate event는 새 snapshot을 만들지 않는다.

## 6. Capability와 session 규칙

### 6.1 환경 변수

~~~text
LIVE_OBSERVATION_POC_ENABLED=false
LIVE_OBSERVATION_POC_AUDIENCE_ORIGIN=https://<cloudfront-domain>
LIVE_OBSERVATION_POC_CURRENT_KID=poc-20260711
LIVE_OBSERVATION_POC_CURRENT_SECRET=<32-byte unpadded base64url>
~~~

LIVE_OBSERVATION_POC_ENABLED 기본값은 반드시 false다. production task definition에는 secret 원문이 아니라 secret store reference로만 주입한다.

### 6.2 HMAC 형식

~~~text
HMAC-SHA256(
  currentSecret,
  "sketchcatch:live-observation:poc:v1\\0"
  + observationId + "\\0"
  + tokenVersion + "\\0"
  + expiresAt
)
~~~

credential은 kid.base64url-hmac 형식이다.

- kid는 [A-Za-z0-9_-]{1,32}만 허용한다.
- secret은 canonical unpadded base64url이며 decode 후 정확히 32 byte여야 한다.
- credential MAC은 decode 후 정확히 32 byte이고 재-encode 값이 원문과 같아야 한다.
- 비교는 timingSafeEqual을 사용한다.
- session metadata에는 kid, tokenVersion, createdAt, expiresAt만 보관한다. token 원문과 hash index는 보관하지 않는다.
- 세션은 생성 시점부터 15분이며, 요청/stream/receipt가 TTL을 연장하지 않는다.

## 7. Audience frontend 동작

apps/web/public/live-observation-poc/index.html은 작은 화면만 제공한다.

~~~text
제목: Live Traffic Test
설명: 버튼을 누르면 CloudFront → ALB → EC2 요청 결과를 SketchCatch에 기록합니다.
버튼: 요청 보내기
상태: aria-live="polite"
~~~

app.js는 다음 순서로 동작한다.

1. location.hash에서 observationId, collector, capability를 읽는다.
2. 값이 없거나 중복되거나 format이 틀리면 버튼을 disabled하고 오류를 보인다.
3. history.replaceState(null, "", location.pathname)으로 fragment를 즉시 지운다.
4. capability는 module scope memory에만 둔다. Storage API와 console.log를 쓰지 않는다.
5. 버튼 클릭 중에는 버튼을 disabled한다.
6. fetch("/api/traffic", { method: "POST", cache: "no-store" })를 호출한다.
7. 2xx, JSON Content-Type, eventId/instanceId/receivedAt schema가 모두 유효할 때만 collector POST를 호출한다.
8. collector에는 Authorization: LiveObservation <capability>를 보낸다.
9. traffic 실패 시 collector를 절대 호출하지 않는다.
10. traffic 성공/collector 실패는 서로 다른 문구로 표시한다.
11. 완료 또는 오류 뒤 버튼을 다시 활성화한다.

성공 문구 예시:

~~~text
EC2 i-0123456789abcdef0에서 요청을 처리했고 SketchCatch에 기록했습니다.
~~~

collector 실패 문구 예시:

~~~text
EC2 요청에는 성공했지만 SketchCatch 기록에 실패했습니다. 다시 시도하세요.
~~~

## 10. 실제 제품으로 승격할 때의 교체 지점

| PoC | 제품 승격 시 교체 |
| --- | --- |
| in-memory ObservationStore | Redis fail-closed atomic LiveObservationStore |
| PoC HMAC key 하나 | current/previous rotation이 있는 capability keyring |
| browser receipt self_reported | 필요 시 EC2 signed receipt + AWS observer evidence |
| PoC session route | Deployment v2 manifest candidate/readiness route |
| manual k6 | 승인된 presenter boost lease와 operation runbook |
| public ALB 임시 타협 | CloudFront origin-facing prefix list, private S3/OAC, restrictive IAM |

최종 Live Observation은 빠른 browser receipt와 늦지만 권위 있는 CloudWatch/ASG/Target Health 관측을 서로 다른 줄과 상태로 보여야 한다. 이 PoC는 그중 첫 번째 줄의 실제 연결만 검증한다.
