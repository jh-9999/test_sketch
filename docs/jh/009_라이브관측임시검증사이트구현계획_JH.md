# 라이브 관측 임시 검증 사이트 실행 문서

이 문서는 Live Observation PoC의 로컬 검증과 승인 후 수동 AWS 검증을 분리한다. 실제 URL, capability, 토큰, AWS 계정 ID는 문서, shell history, CI log에 남기지 않는다.

## 로컬 검증

아래 smoke는 외부 서버를 열거나 호출하지 않는다. 안전한 in-memory store, 고정된 로컬 capability key, 실제 Fastify `inject` 경계로 session 생성, audience fragment 생성, traffic receipt, collector, SSE snapshot, duplicate, expiry를 확인한다. 성공 출력은 일반적인 한 줄뿐이며 fragment URL이나 capability를 출력하지 않는다.

```bash
pnpm --filter @sketchcatch/api exec tsx ../../scripts/live-observation-poc/local-smoke.mjs
```

## audience S3 패키지

audience 정적 자산은 일반 web build와 별도다. 아래 명령은 `apps/web/public/live-observation-poc`의 필요한 정적 파일과 보안 소스 제약을 검사한 뒤, git-ignore된 `dist/` 아래 local tar.gz 하나만 만든다. AWS, HTTP, S3 upload 호출은 하지 않는다.

```bash
node scripts/live-observation-poc/package-audience.mjs
tar -tzf dist/live-observation-poc-audience.tar.gz
```

artifact는 승인된 별도 배포 절차에서만 사용한다. 이 문서와 package script에는 AWS CLI upload 명령을 두지 않는다.

## Terraform 사전 확인

아래는 형식, provider 초기화, validate, plan을 위한 정확한 명령이다. `plan` 결과와 bundle URL은 검토 대상이며, 이 단계에서 AWS `apply`나 `destroy`를 실행하지 않는다.

```bash
terraform -chdir=infra/aws/live-observation-poc fmt -check -recursive
terraform -chdir=infra/aws/live-observation-poc init -backend=false
terraform -chdir=infra/aws/live-observation-poc validate
terraform -chdir=infra/aws/live-observation-poc plan
```

AWS `apply`, `destroy`, S3 upload, 또는 k6 실행은 사용자가 명시적으로 승인한 뒤에만 수행한다. 승인 전에는 실행하지 않는다.

## 승인 후 수동 AWS 확인 목록

승인을 기록한 운영자만 다음을 수동으로 수행한다.

1. 검토한 Terraform plan과 승인된 traffic bundle로 필요한 인프라 적용 및 별도 audience artifact 배포를 수행한다. capability, fragment URL, 계정 ID를 복사해 문서나 로그에 남기지 않는다.
2. 인증된 application session 생성으로 받은 audience fragment 링크를 CloudFront audience 페이지에서 연다.
3. 버튼을 한 번 클릭하고 SSE의 `acceptedEventCount`가 증가하는지 확인한다.
4. `lastReceipt.instanceId`가 `local-dev`가 아니라 EC2 instance ID인지 확인한다.
5. 승인된 load window에서만 다음 형식으로 5분 k6를 실행한다. `COLLECTOR_URL`은 축약된 API origin이 아니라 이미 생성한 session의 complete public event endpoint여야 한다. 이는 collector route에 observation ID가 필요하기 때문이다.

   ```bash
   BASE_URL=https://<cloudfront-domain> \
   COLLECTOR_URL=https://<api-origin>/api/live-observation-poc/public/<observation-id>/events \
   COLLECTOR_AUTH='LiveObservation <capability>' \
   k6 run load/live-observation-poc.k6.js
   ```

6. ALB request metric 증가, ASG desired capacity의 `1 → 2` 증가, 새 target의 healthy 상태를 차례로 확인한다.
7. cleanup도 사용자 명시 승인 후에만 수행한다. 승인 없이 AWS `destroy`를 실행하지 않는다.

이 작업에서는 AWS `apply`, `destroy`, CLI upload, 또는 k6 load를 실행하지 않는다.
